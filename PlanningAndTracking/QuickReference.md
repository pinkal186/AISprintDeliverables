## 🎯 Core Principles

- **Problem → Solution**: Understand need before building
- **Fail Productively**: Build simple baseline to learn constraints  
- **Self-Explain**: Know why each decision exists
- **Research First**: Read original papers, not summaries
- **Build → Test**: Construction before observation
- **Transfer**: Apply to new contexts to prove understanding

## ✅ Quality Checklist

- [ ] Screen + voice recordings submitted
- [ ] AI/plagiarism detector passed
- [ ] No AI-generated images or slop language
- [ ] Evidence-based claims with sources
- [ ] Can defend architecture and decisions
- [ ] Measured failures, not assertions
